# Data model

This document captures the canonical shape of every record stored in the local SurrealDB. It mirrors `../MYCELIUM.md` §8 with the concrete schema migrations applied by `sidecars/surreal_port`.

## Namespace and database

- Namespace: `mycelium`
- Database: `local`

There is exactly one namespace and one database per Mycelium device. Multi-ring support (post-1.0) will use additional namespaces.

## M0 schema

Single table. All other tables (edges, attachments, vector index, op log) are M1+ work.

### `node`

```surql
DEFINE TABLE node SCHEMAFULL;
DEFINE FIELD id          ON node TYPE string;                      -- ULID
DEFINE FIELD kind        ON node TYPE string ASSERT $value INSIDE ["note"];
DEFINE FIELD title       ON node TYPE string DEFAULT "";
DEFINE FIELD body        ON node TYPE array<object> DEFAULT [];   -- list of blocks
DEFINE FIELD created_at  ON node TYPE datetime DEFAULT time::now();
DEFINE FIELD updated_at  ON node TYPE datetime DEFAULT time::now();
DEFINE FIELD schema_ver  ON node TYPE int DEFAULT 1;
DEFINE INDEX node_updated ON TABLE node FIELDS updated_at;
```

#### Block shape (inside `body`)

Each element of the `body` array is an object:

```json
{
  "id": "<ulid>",
  "kind": "paragraph" | "heading" | "bullet",
  "level": 1 | 2 | 3 | 4,            // only when kind == "heading"
  "text": "<utf-8 string>"
}
```

The block schema is intentionally minimal in M0. Marks (bold/italic), nested lists, embedded media, tables, and code blocks land in M1 once the editor is rewritten on top of Loro tree containers.

## Forthcoming tables (M1+)

### `edge` (M1)

Typed relations between nodes. CRDT-managed.

```surql
DEFINE TABLE edge SCHEMAFULL;
DEFINE FIELD id        ON edge TYPE string;
DEFINE FIELD source    ON edge TYPE string;          -- node id
DEFINE FIELD target    ON edge TYPE string;          -- node id
DEFINE FIELD relation  ON edge TYPE string;          -- e.g. "links_to", "tagged"
DEFINE FIELD properties ON edge TYPE object DEFAULT {};
DEFINE FIELD created_at ON edge TYPE datetime DEFAULT time::now();
DEFINE FIELD tombstoned ON edge TYPE bool DEFAULT false;
DEFINE INDEX edge_source ON TABLE edge FIELDS source;
DEFINE INDEX edge_target ON TABLE edge FIELDS target;
```

### `attachment` (M1)

Content-addressed blobs.

```surql
DEFINE TABLE attachment SCHEMAFULL;
DEFINE FIELD blake3     ON attachment TYPE string;          -- BLAKE3 hex
DEFINE FIELD size       ON attachment TYPE int;
DEFINE FIELD mime       ON attachment TYPE string;
DEFINE FIELD created_at ON attachment TYPE datetime DEFAULT time::now();
-- The actual bytes live on disk under <attachments_path>/<blake3-prefix>/<blake3>.age
```

### `op_log` (M1)

Per-document append-only log of CRDT operations.

```surql
DEFINE TABLE op_log SCHEMAFULL;
DEFINE FIELD id        ON op_log TYPE string;
DEFINE FIELD doc_id    ON op_log TYPE string;
DEFINE FIELD lamport   ON op_log TYPE int;
DEFINE FIELD author    ON op_log TYPE string;       -- device id (BLAKE3 prefix of node-pub)
DEFINE FIELD dek_id    ON op_log TYPE int;
DEFINE FIELD ciphertext ON op_log TYPE bytes;       -- age-encrypted Loro op
DEFINE INDEX op_log_doc_lamport ON TABLE op_log FIELDS doc_id, lamport;
```

### Vector index (M2)

Embeddings stored as `array<float, 384>` with HNSW indexing.

```surql
DEFINE TABLE block_embedding SCHEMAFULL;
DEFINE FIELD block_id ON block_embedding TYPE string;
DEFINE FIELD vector   ON block_embedding TYPE array<float, 384>;
DEFINE FIELD model_id ON block_embedding TYPE string;
DEFINE INDEX block_embedding_hnsw ON TABLE block_embedding FIELDS vector HNSW DIMENSION 384 DIST COSINE M 16 EFC 200;
```

## Migrations

Schema versions are tracked in the per-record `schema_ver` field and in a `_meta` table. Each migration script lives in the sidecar code; migrations are forward-only. The application refuses to start if the on-disk schema version is newer than the application supports (preventing silent data loss).

The migration runner is `Storage::migrate` in `sidecars/surreal_port/src/main.rs`. It is called once at startup by `apps/core/src/mycelium/storage/storage_supervisor.gleam`, before any document process is allowed to start.

## ULIDs

Every node and edge identifier is a ULID (Universally Unique Lexicographically Sortable Identifier):

- 128 bits total: 48-bit timestamp (ms since epoch) + 80-bit randomness.
- Encoded in Crockford base-32, 26 characters.
- Sorts lexicographically by creation time.

Generated by `apps/core/src/mycelium/util/ulid.gleam`. ULIDs are time-prefixed but **not** Lamport timestamps; M1's CRDT layer introduces a separate Lamport clock for ordering operations.

## Tombstones (M1+)

Deletion is tombstoning, not erasure: a CRDT op marks the record as deleted; subsequent operations referencing the deleted record are still applied (idempotent on a tombstoned record). Compaction (M3) garbage-collects tombstones older than the stable version vector.
