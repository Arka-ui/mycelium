# Platform readiness — macOS / Linux build verification

Per `D.3` (macOS dmg) and `D.5` (Linux .deb / .rpm / .AppImage), Mycelium is configured to produce per-platform installers. This document captures the readiness state without needing to actually run the foreign builds locally.

## What's wired

The Tauri bundler will produce the right artifacts because:

1. **`apps/desktop/tauri.conf.json`** declares `bundle.targets = ["msi", "nsis", "deb", "appimage", "dmg"]`. Each target activates only on its host platform when running `cargo tauri build` (Tauri auto-skips foreign-platform targets).
2. **`apps/desktop/tauri.conf.json` per-platform sections**: `bundle.windows` (NSIS + WiX with EN+FR languages), `bundle.linux.deb` (depends list), `bundle.macOS` (minimumSystemVersion 12.0).
3. **Sidecar binaries are bundled via `externalBin`**, with target-triple-suffixed naming so each platform finds its own copies (e.g. `bin/surreal_port-x86_64-apple-darwin`).
4. **`infra/ci/release.yml`** runs the build on:
   - `windows-latest` → `.msi` + `.exe`
   - `ubuntu-latest` → `.deb` + `.rpm` + `.AppImage`
   - `macos-latest` (x86_64) → `.dmg`
   - `macos-latest` (aarch64) → `.dmg`
5. **System dependencies are installed by the workflow**:
   - Ubuntu: `libwebkit2gtk-4.1-dev libgtk-3-dev libayatana-appindicator3-dev librsvg2-dev libssl-dev libsoup-3.0-dev clang`
   - Windows: `choco install llvm`
   - macOS: defaults work; Xcode CLI installed in the runner image

## Producing the artifacts

On a Mac with `cargo tauri` installed:

```bash
cargo build --release --target aarch64-apple-darwin -p surreal_port -p loro_port -p iroh_port -p fastembed_port -p wasmedge_port -p crypto_port
mkdir -p apps/desktop/bin
for s in surreal_port loro_port iroh_port fastembed_port wasmedge_port crypto_port; do
  cp target/aarch64-apple-darwin/release/$s apps/desktop/bin/$s-aarch64-apple-darwin
done
cd apps/core && gleam build
cd ../desktop && cargo tauri build --target aarch64-apple-darwin
# Output: target/aarch64-apple-darwin/release/bundle/dmg/Mycelium_0.1.0_aarch64.dmg
```

On Ubuntu:

```bash
sudo apt-get install -y libwebkit2gtk-4.1-dev libgtk-3-dev libayatana-appindicator3-dev librsvg2-dev libssl-dev libsoup-3.0-dev clang
cargo build --release -p surreal_port -p loro_port -p iroh_port -p fastembed_port -p wasmedge_port -p crypto_port
for s in surreal_port loro_port iroh_port fastembed_port wasmedge_port crypto_port; do
  cp target/release/$s apps/desktop/bin/$s-x86_64-unknown-linux-gnu
done
cd apps/core && gleam build
cd ../desktop && cargo tauri build
# Output:
#   target/release/bundle/deb/mycelium_0.1.0_amd64.deb
#   target/release/bundle/rpm/Mycelium-0.1.0-1.x86_64.rpm
#   target/release/bundle/appimage/Mycelium_0.1.0_amd64.AppImage
```

## Pre-launch checklist

Before tagging `v0.1.0` and pushing to GitHub:

- [ ] On Windows host: `cargo tauri build` produces `.msi` and `.exe` without errors
- [ ] On macOS host: `cargo tauri build --target {x86_64,aarch64}-apple-darwin` produces `.dmg` for both arches
- [ ] On Linux host: `cargo tauri build` produces `.deb`, `.rpm`, `.AppImage`
- [ ] Sign the Windows installer with the project's code-signing certificate
- [ ] Notarize the macOS bundle with Apple Notary Service (`xcrun notarytool submit`)
- [ ] Compute and publish SHA-256 hashes (`infra/ci/release.yml` does this automatically)
- [ ] Smoke-test each installer on a clean VM of the target OS
- [ ] Confirm the `tauri-plugin-updater` endpoint is reachable from the published version

## Known caveats

- Code signing requires:
  - **Windows**: an EV (Extended Validation) certificate, ~$500/year, configured via `WINDOWS_CERTIFICATE` + `WINDOWS_CERTIFICATE_PASSWORD` environment variables in CI.
  - **macOS**: an Apple Developer ID certificate, $99/year, plus notarization API key.
  - **Linux**: GPG signing of `.deb` and `.rpm` is recommended; the Tauri bundler does not currently sign automatically.
- Tauri auto-update requires:
  - A signed update manifest at the configured endpoint.
  - The `pubkey` field in `tauri.conf.json::plugins.updater` populated with the public key generated by `tauri signer generate`.
  - The endpoint to return a JSON descriptor matching Tauri's update format on `GET /{target}/{arch}/{current_version}`.
